# אפיון מערכת בסיסית למענה בוואטסאפ בלי Make

## מטרה
להקים מערכת פשוטה וזולה לעסק, שמקבלת פניות מוואטסאפ, מחזירה מענה ראשוני אוטומטי, שואלת כמה שאלות קבועות, ובמידת הצורך מעבירה את השיחה אליך להמשך טיפול ידני.

המערכת מיועדת לעסק עם נפח פניות קטן, בערך 1–3 פניות ביום, ולכן המטרה היא:
- להימנע ממערכות כבדות ויקרות
- להקים משהו שקל להבין ולתחזק
- לא לשלם על אוטומציה מיותרת
- להשאיר אפשרות לשדרוג בעתיד

---

## מה אני ממליץ מקצועית
לעסק קטן כמו שלך, **לא צריך Make בשלב הזה**.

הפתרון הנכון הוא:

**WhatsApp Business Platform (Cloud API) + שרת קטן משלך + OpenAI API + מסד נתונים פשוט**

כלומר:
1. הלקוח שולח הודעה לוואטסאפ העסקי
2. webhook מקבל את ההודעה בשרת שלך
3. השרת בודק אם זו הודעה חדשה / לקוח קיים
4. השרת מחליט מה לענות:
   - תשובת פתיחה
   - שאלות איסוף פרטים
   - תשובה מתוך מאגר מידע
   - העברה אליך
5. השרת שולח תשובה חזרה דרך WhatsApp API
6. כל השיחה נשמרת ללוג / מעקב

## מבנה המערכת

### רכיב 1: Webhook Receiver
אחראי לקבל הודעות נכנסות מ-WhatsApp.

תפקידים:
- לקלוט הודעה חדשה
- לזהות מספר שולח
- לבדוק אם זו הודעה נכנסת אמיתית
- לשמור את ההודעה במסד

---

### רכיב 2: Message Router
אחראי להחליט מה לעשות עם ההודעה.

לוגיקה לדוגמה:
- אם זו הודעה ראשונה → שלח הודעת פתיחה
- אם חסר שם → בקש שם
- אם חסר סוג אירוע → שאל
- אם חסר תאריך → שאל
- אם הלקוח מבקש מחיר → שלח נוסח מתאים
- אם הלקוח כותב משהו מורכב → העבר אליך

---

### רכיב 3: AI Helper
רכיב אופציונלי שמבצע משימות חכמות.

דוגמאות:
- לזהות אם מדובר בבר מצווה / בת מצווה / יום הולדת
- להפוך טקסט מבולגן לפרטים מסודרים
- לנסח תשובה מנומסת
- לסכם את הפנייה במשפט אחד

הערה מקצועית:
AI צריך להיות **עוזר**, לא מנהל המערכת.

---

### רכיב 4: Outbound Sender
אחראי לשלוח הודעות חזרה ללקוח דרך WhatsApp API.

---

### רכיב 5: Admin View בסיסי
לא חובה בהתחלה, אבל מאוד כדאי בהמשך.

מה אפשר לראות:
- רשימת פניות
- סטטוס כל פנייה
- ההודעות האחרונות
- האם צריך לחזור ללקוח
- תיוג לפי סוג אירוע

בהתחלה אפשר גם בלי ממשק, ולשמור הכל ב-Supabase.
בהמשך אפשר לבנות דשבורד קטן.

---

## מה המערכת צריכה לדעת לענות

המערכת לא צריכה להיות חכמה מדי.
היא צריכה להיות מדויקת.

### מומלץ להתחיל מ-4 סוגי תגובות:

#### 1. הודעת פתיחה
לדוגמה:
"שלום, תודה שפנית ל'מצגת – זה כל הסיפור'. אשמח לעזור. כדי להבין מה הכי מתאים, כתוב לי בבקשה איזה אירוע אתם חוגגים ומה התאריך."

#### 2. איסוף פרטים
המערכת שואלת:
- סוג האירוע
- שם החוגג
- גיל
- מועד האירוע
- האם יש תמונות וסרטונים מוכנים
- האם רוצים גם AI / שיר / פתיח מיוחד

#### 3. תשובות FAQ
למשל:
- כמה זמן לוקח להכין מצגת
- איך שולחים חומרים
- מה המחיר
- האם אפשר לשלב AI
- האם אפשר לקבל מהר

#### 4. העברה לטיפול ידני
כשיש שאלה מורכבת, המערכת עונה למשל:
"קיבלתי, זה משהו שכדאי שנבדוק אישית כדי לתת לך תשובה מדויקת. ניצן יחזור אליך בהמשך."

## ארכיטקטורה מומלצת

```text
Customer WhatsApp Message
        ↓
WhatsApp Cloud API
        ↓
Webhook (Node.js / Express)
        ↓
Message Router
   ├── Rules Engine
   ├── OpenAI Helper
   └── Database Check
        ↓
Response Generator
        ↓
WhatsApp Send Message API
        ↓
Customer gets reply
```
